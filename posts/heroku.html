What I learned about deploying to Heroku 22 03 2014

home

Work in progress I wanted to see how to actually deploy an app or a page to a cloud service provider. I went with heroku and quickly ran into a lot of problems - which could easily have been avoided. So here are some practical tips for deployment.

Get an account at Heroku

Got to heroku.com and register for an account. But once done, don't create an app yet - we'll build it using git later (this makes the proces as a whole much easier).

Get the Toolbelt

You need to be able to run heroku commands from the console. In order to do this, you should download the herouku toolbelt. Follow the download instructions and then look at the next section below.

Static or dynamic?

Heruko was build for running serverside apps, i.e. dynamic pages. To make a static page work at heruko you have to make it think the page is dynamic. The easy fix for this is to include a index.php file which points to the main html page of your site.

The proces is explained really well at this resource: "How to deploy a static page to heruko the easy way" by lemiffe.

Using git and github for deployment

If you're not familiar working with git, this is a really great guide, made by Git and Code School. It only takes about 15 minutes and is well worth it.

Once you've gone through the guide, get an account at github.

Next, fire up your console. Once started, use the cd command to switch to the folder you want to stage to git. Then type in the commands to add a new local git repo. Stage, commit - but don't push to git just yet. Your commands should look lise this ($ representing your prompt):

$ git init // initialize this folder as a repo
$ git add . // stage all changes
$ git commit -am "Your message here" // commit changes
// remember: don't push changes to git just yet

Now it's time to make a heroku app and deploy your git repo to it:

$ heroku create // create new app at heruko
$ git push heroku master // pushes the master branch to heroku

Congrats! You're done! If you run into uanticipated problems, check out this guide by heroku on using git with heroku.
